What

Introduction

SUIT(Smart UI Test) is an automation tool supports UI testing in Desktop, Mobile and Web applications. What

Technologies Supported

Different technologies are used to build SUIT. Desktop technologies like WPF, WinForms, Visual Basic 5.0 and above, MFC, ActiveX, Infragistics, Telerik, Stringray, Visual C++, Galaxy (C/C++), Power Builder, QT6 are used by SUIT. Web technologies used by SUIT are HTML 5, Angular, React, Kendo UI, Oracle Apex, ASP .NET, Ajax, PHP, Silverlight, Azure. IOS and Android Mobile platforms are also used.

Features

There are many features supported by SUIT. They are Smart Recorder, Code Snippet support for C#, Image Comparison, GUI Synchronization, Test Management Tool Integration, Reusable Shared Components, Data driven testing, Highlighting the controls, Locate the control, XML based tests, Different ways to synchronize steps, Ordered test execution, Step generator, Mini suit for test labs, Sub-item explorer for trees or grids, Smart recovery for pop ups, Smart clean-up for failed tests.

SUIT Tests Structure

When user create a project in SUIT, the project will be saved in the following format.

TestContextNode à TestSuiteNode à TestSetNode à TestScriptNode.

Actions

Add

This button is used to add control by using bounding rectangle.

Add As Image

The button will be used if user wants to add control as image.

Locate

Locate is used to locate the SUIT repository control that is added. If user clicks on Locate and places bounding rectangle on the control, SUIT will locate the control in the Repository section.

Locates the image control added to SUIT. Python is the prerequisite to use this feature.
